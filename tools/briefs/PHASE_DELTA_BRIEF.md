### A.3 -- Lesson #N19 (new)

```
#### Lesson #N19 -- Analyzer detection via canonical FQN strings, not type references

The analyzer holds no compile-time reference to the assemblies it inspects (no
ProjectReference from the analyzer project to Contracts or any src assembly).
Type identity is matched by canonical fully-qualified display string --
symbol.ToDisplayString() compared against a documented constant (e.g.
"DualFrontier.Contracts.Analyzer.ReservedStubAttribute"). This keeps the analyzer
netstandard2.0-clean, free of load-order and versioning coupling, and makes every
identity the detection depends on a greppable, reviewable string.

**Origin**: A'.9.1 Phase beta design (2026-07-01) -- the W2 FQN-STRING detection
```
